Flick scrolling

An experiment in mobile UX

Introduction

Due to the size constraints of mobile devices, a number of methods for displaying content have evolved:

Continuous Scoll

This is congruent with how content is displayed on standard desktops &emdash; as a long scroll, the viewing position of which can be moved vertically.

Paged Content

This has seen a number of different implementations over the years, but the basic concept is arranging the content into smaller chunks which are navigated using a horizontal interface.

Goals

The objective of this experiment is to provide a jquery library that provides an improved interface for viewing content.

Hypothesis

I believe that these two methods are flawed.

  1. Continous Scroll requires the user to move the content up and down in order to view more content. It provides this interface in a non-structured way, so the user is required to maintain their reading position for the duration of the scroll, making large or poorly formatted documents difficult to work with.
  2. Paged Content tends to use horizontallyâ€“orientated controls. This requires an unnatural gesture for most users. It's a lot easier to move a thumb up and down, instead of left and right on a mobile device.

By combining aspects of the two approaches, we can remove the short comings of both.

Using the vertical gestures of the Continuous Scroll and the structure of Paged Content, we can provide less intensive physical experience. The thumb gestures are simpler, and by displaying content in a consistent place we reduce the amount of work the reader must do to maintain place.
